Technologies Used

HTML Part of Our Website

✱ Our HTML code follows modern web development practices by using semantic elements like <header>, <nav>, <main>, <section>, and <footer> to organize content clearly. It includes important metadata, like responsive design with the viewport setting, UTF-8 character encoding, and links to external resources such as CSS for styling, JavaScript for interactivity, and a favicon for branding.

✱ We have added a user-friendly navigation menu with lists and links to make browsing easier. Images are paired with captions for better context, and alt attributes are used to improve accessibility. We have also combined external resources and inline styles to create a functional, creative, and responsive design while following modern accessibility and usability standards.

CSS Part of Our Website

✱ Our CSS focuses on creating a clean, organized, and responsive design for the website. First, we set some basic styles like removing margins and choosing a consistent font for the whole site. We also defined some custom colors that we use throughout the website to keep things looking cohesive.

✱ We made sure the site works well on both large screens and smaller devices by using media queries. For example, when the screen size gets smaller, the navigation menu changes from horizontal to vertical, and the layout of sections like the team info adjusts to fit better on mobile.

✱ In the team and project sections, we used flexbox to create neat grids that adjust automatically. We added hover effects to make the cards interactive, like making the images slightly darker or adding a shadow when you hover over them.

✱ The navigation bar is easy to use with hover effects, which change the background color, and we also highlighted the active link so users know where they are. In the footer, we used flexbox to align everything and made sure it looks good on smaller screens by stacking the content vertically.

✱ For the technologies section, we made the images interactive by slightly enlarging them and darkening them when hovered over. There's also a slideshow that lets users cycle through images with smooth transitions, and we added next/previous buttons for easy navigation.

✱ Finally, we styled the paragraphs for better readability, adjusting font sizes and spacing. We used the border-box rule to make sure everything sizes correctly, keeping the layout consistent.

JavaScript Part of Our Website

✱ The JavaScript in the project controls a slideshow, adds interactivity to the team name, and draws on a canvas.

✱ The slideshow functionality works by showing the current slide based on an index (slideIndex). The showSlides(n) function is responsible for displaying the correct slide, hiding others, and updating the active dot. The plusSlides(n) function is used to navigate between slides, either forward or backward, by changing the slideIndex and calling showSlides to refresh the view. Similarly, the currentSlide(n) function directly sets the slideIndex to a specific slide when a dot is clicked, displaying the relevant slide.

✱ In the case of the team name, the script listens for a mouseover event on the element with the class team-name. When hovered over, it appends " (Team 20)" to the team name, but only once.

✱ Finally, there is a canvas element with the ID art-canvas. If this element is present, the script draws a line on the canvas when the page is loaded.

In short, the JavaScript enhances the page by managing the slideshow, adding interactivity to the team name, and drawing on the canvas.
